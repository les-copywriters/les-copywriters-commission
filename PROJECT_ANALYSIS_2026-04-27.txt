LES COPYWRITERS COMMISSION
Project Analysis
Date: 2026-04-27

==================================================
1. PROJECT OVERVIEW
==================================================

This project is a React + TypeScript + Vite web application backed by Supabase.
It is a commission operations dashboard for a sales organization.

The application combines:
- commission tracking
- refunds / unpaid monitoring
- team management
- sales analytics
- closer coaching / call analysis
- setter operations metrics
- AI-assisted sales coaching
- automated multi-platform sync workflows

Main external platforms integrated in the codebase:
- JotForm
- Aircall
- iClosed
- Fathom
- Google Gemini (for AI analysis / framework generation / assistant behavior)
- GitHub Actions (for scheduled sync orchestration)

7. DATA MODEL SUMMARY
==================================================

Main business tables visible from the codebase:
- profiles
- sales
- refunds
- impayes
- bonus_tiers
- call_analyses
- closer_frameworks
- closer_profiles
- assistant_threads
- assistant_messages
- assistant_memory_snapshots
- setter_integration_mappings
- setter_call_metrics_daily
- setter_funnel_metrics_daily
- setter_call_records
- integration_sync_runs
- global_settings

What they represent:

profiles
- application users and role assignment

sales
- central commission transaction table
- contains client, product, closer/setter IDs, HT/TTC amount fields, payment type, commissions, refund/unpaid flags, JotForm submission ID

refunds / impayes
- issue tracking against sales

bonus_tiers
- monthly volume-based bonus thresholds

call_analyses
- imported Fathom meetings + transcripts + AI analysis results

closer_frameworks
- AI-generated markdown frameworks for a closer

closer_profiles
- aggregated coaching profile / strengths / risks / score summaries

setter_integration_mappings
- links setters to Aircall / iClosed identities and credentials

setter_call_records
- detailed imported Aircall calls

setter_call_metrics_daily
- daily rollups from call records

setter_funnel_metrics_daily
- daily rollups from iClosed / funnel activity

integration_sync_runs
- stores operational sync runs for Aircall / iClosed scheduled/manual syncs

assistant_threads / assistant_messages / assistant_memory_snapshots
- sales assistant conversation layer and AI memory

9. SUPABASE EDGE FUNCTIONS
==================================================

Shared modules:
- _shared/setterDashboard.ts
- _shared/closerProfile.ts

Function summary:

aircall-lookup
- fetches Aircall users for mapping/setup

analyze-call
- uses Gemini to score and analyze a call transcript

auto-sync
- lightweight orchestrator for cron sync flow

commission-health-report
- admin-only operational health report for data quality, sync freshness, and reconciliation snapshot

generate-framework
- creates a markdown sales framework from transcripts

iclosed-lookup
- validates iClosed and fetches user data for mapping/setup

invite-user
- creates/invites a user into the system

jotform-webhook
- webhook ingestion path for JotForm submissions

refresh-closer-profile
- rebuilds AI closer profile from call-analysis data

sales-assistant
- AI assistant using stored sales/call/coaching context

schedule-setter-dashboard-sync
- scheduled Aircall + iClosed sync entry point

sync-fathom
- imports Fathom meetings/transcripts into call_analyses

sync-jotform
- paginated JotForm sync into sales

sync-setter-dashboard
- manual or admin-triggered setter dashboard sync / validation

==================================================
16. CURRENT LIMITATIONS / RISKS
==================================================

Based on the code review, important limitations include:

1. The system is integration-heavy
- reliability depends on external APIs and credentials staying healthy

2. Health checks are improving but not yet full live reconciliations
- the current admin health report summarizes internal system state and sync freshness
- it does not fully live-compare every source API on each run

3. A lot of business logic lives in Edge Functions
- operational debugging often depends on Supabase logs and external API behavior

4. The frontend is feature-rich and broad
- continued work will benefit from keeping logic modular and tested

18. BOTTOM-LINE SUMMARY
==================================================

This project is a sales operations and coaching platform centered on commissions.

It is not just a commission tracker.
It currently contains:
- commission accounting
- team operations
- setter performance monitoring
- closer call intelligence
- AI-generated coaching and assistant features
- multi-platform ingestion pipelines
- admin diagnostics and sync monitoring

Architecturally, it is a single frontend app with a fairly rich Supabase backend layer and a large amount of business logic in Edge Functions.
